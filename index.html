Labyr - Java Game Engine

This game was a course project for my Java course in University. The game is still not finished because the capacity of this project was way too optimistic due to the short course length and my insufficient skills back then. Nevertheless, this is definitely the project that i’m most proud of. I'm glad i chose this game as a project because it quickly taught me a lot about programming fundamentals. I decided to write almost everything from scratch and therefore i only used standard java libraries and wrote my own engine for the game.

The game was intended to be top-down rougelike game, where each level would be a labyrinth. In the labyrinth you face different obstacles and enemies you have to kill. In the end of each level/labyrinth there's a boss fight.

Even though i didn't finish the game, the game engine is in state where continuing wouldn't be too hard. I wrote a proper tile and sprite system for the game and made loading textures/sprites easy. Levels can be designed in MS Paint. Each pixel represents a tile on the map. Tiles can be differentiated by colors of the pixels.

Currently the game has 1 test level. The only mob right now is the player. The player sprites are animated (walking animations). You can move the player with arrow keys or 'WASD'. I've already implemented collision detection, so you cant walk through solid tiles. I also implemented pixel perfect shooting mechanism to the game. You can shoot little bullets by clicking Mouse1. The bullets also have collision detection with solid tiles which could be improved a little bit. Finally i tried to implement particle effects. I managed to get them working but i haven't finished implementing proper particle physics.

C# & WPF - Car repair garage system

This project was made as a course project for C# course. Since it was a team project, we had to form teams of 3 people. We had to go through full cycle of development. In the beginning we did some research and planning. After that we had to write a proper analysis for the technologies and development patterns that we had chosen. Then we had to distribute the workload and work on the prototype solution as a team. This project gave me an idea how to properly work in a team when developing software.

Basically, it's a mock-up system for car repair garages and their clients to reserve times for different repair services. We used code first development with entity framework. We chose MVVM as architectural pattern. As this was our first course on C#, we faced a lot of challenges with learning the functionality of Visual Studio. It was a great experience.

To download the source code and get more information on the project, visit our projects wiki site. Unfortunately the page is available only in Estonian language.
